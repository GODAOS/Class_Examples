○ 개요
	1. 필수 기능
		- 웹 -
			1.사용자의 개인 정보 작성 및 처방 받은 약 제품명을 입력하면 api를 통하여 나머지 정보 자동 입력.
			2.복용 내역 DB 저장 
			3.증상 입력시 추천 약 제공
			4.구현 할 약품 이미지만 미리 저장하여 발표 때 활용

		-앱-
			1.약통에서 약을 먹으면 앱으로 푸쉬가 옴.
			2.약을 복용하지 않으면 설정된 주기마다 경고창 옴.
			3.주변 약국 및 병원 조회.
			4.증상 입력시 해당하는 약 정보 제공. (ex: 두통 → 타이레놀)

		-IOT-
			1. 약 복용 여부 체크 후 표시
			2. 뚜껑이 열리면 앱에 알림이 뜨고 사용자가 복용 후 직접 앱에서 복용여부 클릭
			3. 버튼이 눌렸을 때 만약 약 시간이 아니면 앱에 알림이 나옴
			4. 블루투스 연결이 끊기면 사용자에게 "약통을 챙겼나요?"알림 주기(안챙길거임 클릭하면 한동안 알람 안뜨게 하기)



		※ 추가 예정) 병원 캘린더(예약 기록), 추천 식품 

○ 스토리 보드
	- 앱 (기한 6월달 안에 끝날 것)- 
		* 앱을 킨다
		* 로고, 앱이름, 빙글빙글
		* 메인 화면
			1) 햄버거 누를 시 로그인 정보
			2) 로그인 정보 밑에 로그인 / 회원가입 버튼
			3) 
		▶필요한 것: 메인 화면 구성, 햄버거 화면 구성
		
		* 햄버거 화면
			1) 각 메뉴의 대분류 - 소분류
		
		* 약 시간이 되면 앱을 키지 않아도 약 먹을 시간 알람 / 확인 버튼
		
		* 상태창에 항상 위젯이 떠있을 것 / 약 먹음 버튼
			5 - 1. 오늘 복용 예정 시각, 약 안내 / 사용자가 메모한 것 출력
			5 - 2. 약 먹기 버튼 / 누르면 가장 가까운 복용 시간 체크되게끔
			5 - 3. 예정이 없으면 오늘 복용 예정이 없습니다.
			
		* 반복 주기, 소리, 소리 크기, 진동, 미루기(n분) 설정
		
		* 알람 1시간이나 30분전에 미리 알림(알람 끄기 버튼, 확인버튼)
		
		* 복용하는 약 사진을 촬영해서 기록하고 몇알이나 남았고 몇알남았을때 알림이 뜨게할지 설정
		
		* 유통기한도 적게끔(선택)
		
	- IoT -
		1. 뚜껑 열기
			1 - 1 : 닫히면 버튼 눌리는걸 이용해서 감지
			1 - 2 : 관절부에 가변저항이나 모터를 달아서 감지
			1 - 3 : 버튼 누르면 열리면서 동시에 열림을 감지
				
		2. 뚜껑 열리면 알람
			2 - 1 : 그냥 열렸다고 알람 / 확인버튼
		
		3. 뚜껑 닫음
		
		4. 사용자가 약통을 놓고갈 경우 알람
			4 - 1 : 약통을 챙기셨나요? ▶ N분 동안 안챙겨도 됨, 확인버튼 
		
		- 추가 부분
	
		5. 약 복용 스케쥴을 잊어버렸을 경우(ex : 약 복용 시간 30분이 지나도 알람이 안꺼짐) 지정한 사람(가족, 친구 등)에게 알람
		
		6. 알람이 N분 주기로 미룰수 있는 기능 미룰수 있는 기능 (최대 30분)
		
		7. 뚜껑이나 따로 부착한 버튼을 치면 알람이 무음으로 변경
